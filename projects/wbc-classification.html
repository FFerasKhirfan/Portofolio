White Blood Cell Classification CNN

A deep learning system that classifies different types of white blood cells with high accuracy for faster disease diagnosis.

Python TensorFlow Keras NumPy Pandas OpenCV

Project Overview

The White Blood Cell Classification CNN is a machine learning system I developed to automate and accelerate the diagnosis of blood-related diseases. By leveraging deep learning techniques, the model can identify five different types of white blood cells from microscopic images, providing valuable diagnostic information to healthcare professionals.

The system uses a TensorFlow Keras model (stored in blood_cell_classifier.h5) that I trained specifically for this purpose. The application preprocesses blood cell images by resizing them to 96Ã—96 pixels and normalizing pixel values to a range of 0 to 1, ensuring consistent input for the neural network. I developed a user-friendly GUI using Tkinter that allows medical professionals to easily select and analyze images. The application also includes comprehensive error handling to manage exceptions during image loading or classification, with clear feedback displayed to users.

Key Features

  • 93% classification accuracy across various WBC types (basophil, eosinophil, erythroblast, ig, lymphocyte, monocyte, neutrophil, platelet)
  • Data augmentation techniques to improve model robustness
  • Interpretable model results with visualization of feature activation

Technical Implementation

The project architecture consists of the following components:

1. Data Processing Pipeline

I implemented a robust preprocessing pipeline that handles image normalization, contrast enhancement, and segmentation. This ensures the model can handle images from different microscopes and with varying quality. Data augmentation techniques such as rotation, flipping, and zoom were applied to expand the training dataset and improve model resilience.

2. CNN Architecture

I designed a custom convolutional neural network with 14 layers, including convolutional, pooling, batch normalization, and dropout layers. The architecture was carefully optimized to learn the distinctive morphological features of each cell type while minimizing overfitting. Transfer learning was also explored, with a fine-tuned VGG16 model serving as a comparative baseline.

3. Model Training and Evaluation

The model was trained on a dataset of 10,000+ labeled WBC images, with 5-fold cross-validation to ensure robustness. Hyperparameter optimization was performed using a combination of grid search and Bayesian optimization techniques. Performance was evaluated using metrics including accuracy, precision, recall, F1-score, and confusion matrices.

4. Deployment Interface

I developed a Tkinter-based desktop application that allows medical technicians to upload blood smear images and receive real-time classification results. The interface includes visualization tools that highlight cell features contributing to the classification, providing transparency and aiding in result verification.

5. Distribution & Deployment

To ensure widespread usability, I packaged the application using PyInstaller to create a standalone executable. This approach eliminated the need for end users to install Python or any dependencies, making the tool accessible to healthcare professionals regardless of their technical background. The packaged application includes all necessary model files and libraries while maintaining a small footprint.

Results & Impact

  • 93% overall classification accuracy, comparable to expert hematologists
  • Huge reduction in the time required to analyze blood samples
  • Potential for integration with existing laboratory information systems
